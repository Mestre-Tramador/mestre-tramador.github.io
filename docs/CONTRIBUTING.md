# Contribution Guidelines

**Read also in: [Español], [Português Brasileiro]**

First of all, thank you very much in advance, someone wanting contributing with
such personal project means a lot to me. Also, before scrolling down, check out
the [Code of Conduct] and the [Security Policy], both files have really good info
if you have any doubt.

Below is a simple list of contributions that you can make:

- [Testing]
  - [Bug Reports]
- [Pull Requests]
- [Documentation]
- [Other Contributions]

## Testing

Please, don't be afraid to clone my repository and test it freely! If you found
some unexpected result, please, check below some procedures to do in these cases...

### Bug Reports

If you encounter a bug, please, report it using [this][Bug Report] Issue Template,
describing concisely the behavior of the glitch. Also, I'm really sorry for the trouble.

## Pull Requests

I'm studying at this moment the workflow of the Pull Requests Templates. When I'm
done, all my public repos will be updated to have them, so, currently all Pull
Requests will be **rejected**.

## Documentation

A Wiki will be written as soon as the code take shape, so soon it should be available.
If by any chance you find an error in the docs, checkout [this][Docs Revision]
Issue Template to assign a revision.

## Other Contributions

A star would be gladly received! Also, if you have any questions, make them
[here][Question Answer].
